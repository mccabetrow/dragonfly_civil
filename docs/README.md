## Documentation Standards

1. **File naming:** `lowercase_with_underscores.md`
2. **Versioning:** Include version number in title or header for policy docs
3. **Updates:** Update "Last Updated" date when making significant changes
4. **Review:** Governance docs require CEO signoff before finalizing
